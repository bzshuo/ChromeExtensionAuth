# 步骤记录：安装到纯英文路径以修复 Chrome 不执行

**时间戳**：20260303_120827  
**操作摘要**：安装到纯英文路径

---

## 现象

- manifest 的 path 正确指向当前项目的 `run.sh`。
- 手动在终端执行 `run.sh` 会生成 `/tmp/native_host_jiangtai_debug.log`，说明脚本和 Python 正常。
- 由 Chrome 触发时不会生成该日志，说明 **Chrome 没有真正执行到我们这份脚本**。

## 原因

项目路径中含有中文目录名「脚本」：  
`/Users/bzshuo/Documents/JiangTai/脚本/chrome-extension-jiangtai-auth/...`  
在 macOS 上，Chrome 启动 Native Messaging 主机时对**含非 ASCII 的路径**可能无法正确执行，导致进程未启动或立即退出，且无日志。

## 修改内容

**install_mac.sh**：

- 不再让 manifest 的 `path` 直接指向项目里的 `run.sh`。
- 在 Chrome 的 Native Messaging 目录下创建**纯英文子目录** `jiangtai_auth_host`，将 `run.sh` 和 `write_auth.py` **复制**到该目录。
- manifest 的 `path` 改为指向该副本，例如：  
  `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/jiangtai_auth_host/run.sh`  
  这样 Chrome 启动的路径中不再包含「脚本」等非 ASCII 字符。

## 请你操作

1. **重新执行安装**（把 `你的扩展ID` 换成 chrome://extensions 里本扩展的 ID）：

   ```bash
   cd "/Users/bzshuo/Documents/JiangTai/脚本/chrome-extension-jiangtai-auth/native_host"
   ./install_mac.sh 你的扩展ID
   ```

2. **完全退出 Chrome（Cmd+Q）后重新打开**。

3. **再测**：打开扩展弹窗，看是否显示「Native 主机：已连接」；或访问卫健网触发带 Auth 的请求，看桌面是否生成 `api_tokens.json`。

4. 若仍有问题，可查看日志：  
   `cat /tmp/native_host_jiangtai_debug.log`  
   此时由 Chrome 触发时也应能写入该文件。
