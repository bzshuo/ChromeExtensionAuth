# 步骤记录：弹窗 Native 诊断与步骤文档

**时间戳**：20260303_114206  
**操作摘要**：弹窗增加 Native 主机诊断与步骤记录

---

## 用户需求

- 没有在控制台看到明显报错。
- 现状：浏览器监听到带 Authorization 的接口后直接触发下载文件，桌面没有生成 `api_tokens.json`。

---

## 原因说明

- 扩展会优先通过 **Native Messaging** 把 token 写入桌面指定路径；若连接 Native 主机失败，则**降级为使用 Chrome 下载**，文件会保存到 Chrome 默认下载目录（多为「下载」文件夹），而**不会**在桌面生成文件。
- 因此当前现象说明：**Native 主机未连接成功**，所以一直走下载兜底。

---

## 本次修改内容

1. **弹窗内增加 Native 主机连接状态诊断**
   - 在 `popup.html` 中增加一行：`Native 主机：检测中…`。
   - 打开弹窗时，`popup.js` 向 Native 主机发送一次探测（无实际写入的 merge），根据结果更新文案：
     - **已连接**：显示「Native 主机：已连接（会写入桌面文件）」绿色。
     - **未连接**：显示「Native 主机：未连接 — [Chrome 返回的错误信息]」红色，便于根据提示排查（如扩展 ID 不匹配、未安装主机、未重启 Chrome 等）。
   - 样式：`.native-ok` 绿色、`.native-fail` 红色。

2. **新增步骤文档**
   - 本文件：`docs/steps/20260303_114206_弹窗Native诊断与步骤.md`，记录用户需求与本次修改说明。

   - 终端执行（把 `你的扩展ID` 换成刚复制的）：
     ```bash
     cd "/Users/bzshuo/Documents/JiangTai/脚本/chrome-extension-jiangtai-auth/native_host"
     chmod +x run.sh write_auth.py
     ./install_mac.sh 你的扩展ID
     ```

   - **完全退出 Chrome 后重新打开**（不是只关窗口）。

2. **看弹窗诊断结果**
   - 重新打开扩展弹窗，看「Native 主机」一行：
     - 若为「已连接」：再访问卫健网触发带 Authorization 的请求，桌面应出现/更新 `api_tokens.json`。
     - 若为「未连接 — xxx」：根据 xxx 提示排查（如指定了错误的 Chrome 用户目录、未用系统自带的 Google Chrome 等）。

3. **确认下载目录（仅作验证）**
   - 若仍未连接，当前行为会把 JSON 下载到 Chrome 默认下载目录，可在 Chrome 设置中查看「下载位置」，或打开「下载」文件夹查看是否有 `api_tokens.json`。
