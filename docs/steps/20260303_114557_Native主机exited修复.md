# 步骤记录：修复 Native host has exited

**时间戳**：20260303_114557  
**操作摘要**：Native 主机 exited 修复

---

## 用户反馈

弹窗显示：**Native 主机：未连接 — Native host has exited.**

---

## 原因说明

Chrome 在启动 Native Messaging 主机时，**不会继承用户终端的 PATH**，子进程环境里往往没有 `python3`。  
`run.sh` 里原先写的是 `exec python3 ...`，在 Chrome 调用的上下文中相当于「命令找不到」，进程一启动就退出，Chrome 就会报 “Native host has exited”。

---

## 修改内容

- **文件**：`native_host/run.sh`
- **修改**：将 `exec python3 "$DIR/write_auth.py"` 改为使用 Python 的**绝对路径**  
  `exec /usr/bin/python3 "$DIR/write_auth.py"`
- **说明**：当前机器上 `which python3` 为 `/usr/bin/python3`，故采用该路径。若你本机 Python 来自 Homebrew（如 `/opt/homebrew/bin/python3`），可自行把 `run.sh` 里这一行改成对应路径。

---

## 建议操作

1. 保存修改后，**重新打开扩展弹窗**，看是否显示「Native 主机：已连接」。
2. 若仍为「未连接」，请确认本机默认的 `python3` 路径（在终端执行 `which python3`），把 `run.sh` 里的 `/usr/bin/python3` 改成该路径后，再试一次。
