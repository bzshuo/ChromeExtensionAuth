# 步骤记录：日志改为 /tmp 并排查 manifest

**时间戳**：20260303_115923  
**操作摘要**：日志改为 tmp 并排查 manifest

---

## 用户反馈

没有生成 `native_host_debug.log`（之前日志写在项目目录 `native_host/` 下）。

---

## 原因与修改

Chrome 启动 Native 主机时，工作环境可能无法写入项目目录（权限或路径问题），导致日志文件从未创建。

- **修改**：将日志路径改为 **`/tmp/native_host_debug.log`**，由 `run.sh` 和 Python 统一写到这里。`/tmp` 对当前用户可写，一般不受项目路径影响。

---

## 请你按顺序做

### 1. 确认 manifest 指向当前的 run.sh

Chrome 只会执行 manifest 里填的 `path`。若你移动过项目或重装过扩展，manifest 里可能还是旧路径。

在终端执行：

```bash
cat "$HOME/Library/Application Support/Google/Chrome/NativeMessagingHosts/com.auth_capture.bridge.json"
```

看里面的 **`path`** 是否等于你当前项目里的 `run.sh` 的**完整路径**，例如：

`/Users/bzshuo/Documents/JiangTai/脚本/chrome-extension-jiangtai-auth/native_host/run.sh`

若不一致，需要重新安装 Native 主机（见下）。

### 2. 重新安装 Native 主机（推荐）

在终端执行（把 `你的扩展ID` 换成 chrome://extensions 里本扩展的 ID）：

```bash
cd "/Users/bzshuo/Documents/JiangTai/脚本/chrome-extension-jiangtai-auth/native_host"
./install_mac.sh 你的扩展ID
```

然后**完全退出 Chrome 再重新打开**。

### 3. 再触发一次并看 /tmp 日志

- 打开扩展弹窗，或访问卫健网触发一次带 Authorization 的请求。
- 查看是否生成了日志：

```bash
cat /tmp/native_host_debug.log
```

- 若 **有内容**：把内容发给我，便于继续排查。
- 若 **没有这个文件**：说明 Chrome 没有执行到我们这份 `run.sh`（多半是 manifest 的 path 仍指向别处），请把上面第 1 步里 `cat` 出来的 **整段 JSON** 发给我。
