# 步骤记录：添加 Native 主机调试日志

**时间戳**：20260303_115129  
**操作摘要**：添加 Native 调试日志排查 exited

---

## 用户反馈

修改 run.sh 使用 `/usr/bin/python3` 后，仍报 **Native host has exited**。

---

## 修改内容

1. **run.sh**
   - 将 stderr 追加写入 `native_host/native_host_debug.log`，便于看到 Chrome 启动时的报错。
   - 启动时打印一行 `run.sh started DIR=...`。
   - 增加 Homebrew Python 回退：若 `/usr/bin/python3` 不存在则尝试 `/opt/homebrew/bin/python3`。
   - 通过环境变量 `NATIVE_HOST_DEBUG_LOG` 把日志路径传给 Python。

2. **write_auth.py**
   - 若存在 `NATIVE_HOST_DEBUG_LOG`，则写入调试信息：main 开始、read_message 完成、异常时打印 traceback、main 结束。

---

## 请你操作

1. **再触发一次**：打开扩展弹窗（或访问卫健网触发一次带 Auth 的请求），使 Chrome 再次启动 Native 主机。
2. **查看日志**：打开文件  
   `native_host/native_host_debug.log`  
   （在项目下的 `native_host` 目录里）。
3. **把日志内容发给我**（或贴关键几行）：
   - 若只有 `run.sh started`、没有 `python main() started` → 说明 Python 没起来（路径或权限问题）。
   - 若有 `python main() started` 后有异常 → 看 traceback 定位。
   - 若完全没有新内容 → 可能是 Chrome 没执行到这份 run.sh（例如 manifest 指向了别的目录）。
