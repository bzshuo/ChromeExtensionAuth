## 本次需求

- Chrome 扩展已自动写入 `api_tokens.json`，希望 Apifox 调用接口前自动获取最新 token，避免维护 apifox_env_*.json。

## 本次改动

- 在 `native_host` 下新增 `token_http_server.py`，实现一个本地 HTTP 服务：
  - 监听 `127.0.0.1:5678`，提供 `GET /token?env=dev|test|prod` 接口。
  - 从 `/Users/bzshuo/Documents/JiangTai/Script/api_tokens.json` 读取各环境 token，并以 JSON 返回 `{ env, token }`。
  - 支持通过环境变量 `TOKEN_SERVER_HOST`、`TOKEN_SERVER_PORT`、`API_TOKENS_PATH` 覆盖默认配置。
- 更新 `apifox/前置脚本-读取api_tokens.js`：
  - 去掉对 `fs.readFileSync` 的依赖，改为使用 `pm.sendRequest` 调用本地 `http://127.0.0.1:5678/token?env=...`。
  - 按 Apifox 环境变量 `env`（dev/test/prod）获取对应 token，并设置到 `Authorization` 环境变量。
  - 在文件注释中说明使用步骤：启动本地服务、在环境中配置 `env` 变量、在前置操作中粘贴该脚本。

## 使用说明（简要）

- 终端启动本地服务：

  ```bash
  cd /Users/bzshuo/Documents/JiangTai/Script/chrome-extension-jiangtai-auth
  python3 native_host/token_http_server.py
  ```

- 在 Apifox 中：
  - 为各环境设置 `env = dev/test/prod`。
  - 在项目或接口的「前置操作」中粘贴 `apifox/前置脚本-读取api_tokens.js` 的内容。
  - 确保 Chrome 扩展已正常抓到 Authorization 并写入 `api_tokens.json`，即可在调接口前自动获取最新 token。
