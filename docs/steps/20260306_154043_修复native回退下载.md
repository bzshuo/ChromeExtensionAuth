## 操作摘要

- **时间**: 2026-03-06 15:40:43
- **需求**: 当前版本中，native 可以连接成功，但浏览器仍会下载 token 文件，需要修复。

## 修改内容

- **调整 native 回退逻辑**:
  - 在 `background.js` 的 `writeEnvToken` 中，将原先 `chrome.runtime.sendNativeMessage` 回调里的判断：
    - 从「`chrome.runtime.lastError` 或 `!reply` 或 `!reply.success` 就触发 `fallbackToDownload`」，
    - 修改为仅当存在 `chrome.runtime.lastError` 时才触发 `fallbackToDownload`。
  - 对于没有 `lastError` 但 `reply` 为空或 `reply.success !== true` 的情况，不再触发浏览器下载，而是将返回结果写入 `DEBUG_KEY` 调试日志中，方便在弹窗中排查：
    - 记录 `type: 'nativeError'` 时的错误信息（带时间与错误 message）。
    - 记录 `type: 'nativeReply'` 时的原始 `reply` 内容，用于观察 native 主机返回的结构。

## 预期效果

- **native 主机正常可用** 时：
  - 不再因为 `reply.success` 判定问题导致浏览器额外下载 `api_tokens.json` 文件。
  - 若 native 主机调用失败（如未安装/未授权），依旧会触发下载，保证无 native 环境下的兜底能力。
- **调试信息更清晰**:
  - 通过扩展弹窗中读取调试列表，可以看到 native 调用的错误或异常返回，辅助定位环境问题（例如文件权限、路径错误等）。
